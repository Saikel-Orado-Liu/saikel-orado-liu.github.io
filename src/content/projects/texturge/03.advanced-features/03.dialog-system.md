# 对话系统

**对话控件**（`UDialogWidget`）是基于 `UUserWidget` 和 `UTextAnimator` 构建的完整对话界面方案，面向 RPG、视觉小说和叙事驱动型游戏中的打字机对话展示需求。

## 控件概述

| 属性 | 说明 |
| --- | --- |
| 基类 | `UUserWidget` |
| 动画引擎 | `UTextAnimator`（内置打字机管线） |
| 可选集成 | CommonUI（若定义了 `WITH_COMMONUI` 宏，可在蓝图子类中使用 CommonActivatableWidget 激活栈） |

## API 参考

### 对话内容

```cpp
// 设置对话文本和富文本标签
UFUNCTION(BlueprintCallable, Category = "对话")
void SetDialogText(
    const FText& InText,         // 纯文本内容
    const FString& InRichTags    // 富文本格式标签（含内联动画标签）
);
```

`InRichTags` 参数允许在对话文本中嵌入 Texturge 动画标签标记。实际显示时，标签被解析并驱动对应的动画效果。

### 说话者信息

```cpp
// 设置说话者名称
UFUNCTION(BlueprintCallable, Category = "对话")
void SetSpeakerName(const FText& InName);

// 设置说话者头像
UFUNCTION(BlueprintCallable, Category = "对话")
void SetSpeakerPortrait(UTexture2D* InPortrait);
```

### 动画控制

```cpp
// 使用动画预设启动打字动画
UFUNCTION(BlueprintCallable, Category = "对话")
void StartTyping(
    FName InPresetName,        // 动画预设名称
    float InDuration = 1.0f,   // 动画持续时间（秒）
    float InDelay = 0.0f       // 起始延迟（秒）
);

// 跳过打字动画直接显示全部文本
UFUNCTION(BlueprintCallable, Category = "对话")
void SkipToEnd();

// 检查动画是否正在播放
UFUNCTION(BlueprintPure, Category = "对话")
bool IsTyping() const;
```

### 委托

| 委托 | 签名 | 说明 |
| --- | --- | --- |
| `OnDialogComplete` | `()` | 打字动画完成时触发（蓝图可绑定） |

## 使用模式

### 基本流程

1. 创建继承自 `UDialogWidget` 的蓝图子类
2. 在 UMG Designer 中布局 UI 元素（文本块、头像框、名称栏等）
3. 在游戏逻辑中调用 `SetDialogText()` + `SetSpeakerName()` + `SetSpeakerPortrait()`
4. 调用 `StartTyping()` 开始逐字动画
5. 监听 `OnDialogComplete` 事件，在对话结束后推进剧情

### 交互模式

| 交互 | 行为 |
| --- | --- |
| 等待动画完成 | 自动推进，完成后广播 `OnDialogComplete` |
| 按下确认键 | 调用 `SkipToEnd()` 立即显示全部文本 |
| 按下跳过键 | 广播完成事件并加载下一段对话 |

## 内部实现

`UDialogWidget` 内部通过 `GetOrCreateAnimator()` 懒创建 `UTextAnimator` 实例：

```
SetDialogText()
    │
    ▼
GetOrCreateAnimator()
    │
    ▼
Animator->SetRichText(RenderedRichText)
    │
    ▼
StartTyping()
    │
    ▼
Animator->StartAnimating()
    │
    ▼ (每 Tick)
Animator->TickAnimation(DeltaTime)
    │
    ▼ (完成时)
OnAnimationFinished() → OnDialogComplete.Broadcast()
```

### 多格式文本渲染

对话文本中的 `InRichTags` 参数内容与 `InText` 内容合并后，通过 `SetRichText()` 传入动画器。动画器使用 `FTagParser` 解析所有标签，`FRenderTreeBuilder` 构建渲染树，动画标签被剥离后剩余的标准多格式文本样式标签保留在 `DisplayRichText` 中供 `SAnimatedRichTextBlock` 渲染。
