# Dialog System

The **Dialog Widget** (`UDialogWidget`) is a complete dialog interface built on `UUserWidget` and `UTextAnimator`, designed for RPGs, visual novels, and narrative-driven games requiring typewriter-style dialog presentation.

## Widget Overview

| Property | Description |
| --- | --- |
| Base class | `UUserWidget` |
| Animation engine | `UTextAnimator` (built-in typewriter pipeline) |
| Optional integration | CommonUI (if `WITH_COMMONUI` macro is defined, CommonActivatableWidget activation stack available in Blueprint subclasses) |

## API Reference

### Dialog Content

```cpp
// Set dialog text and rich text tags
UFUNCTION(BlueprintCallable, Category = "Dialog")
void SetDialogText(
    const FText& InText,         // Plain text content
    const FString& InRichTags    // Rich text format tags (including inline animation tags)
);
```

The `InRichTags` parameter allows embedding Texturge animation tag markup within dialog text. Tags are parsed at display time and drive the corresponding animation effects.

### Speaker Information

```cpp
// Set speaker display name
UFUNCTION(BlueprintCallable, Category = "Dialog")
void SetSpeakerName(const FText& InName);

// Set speaker portrait
UFUNCTION(BlueprintCallable, Category = "Dialog")
void SetSpeakerPortrait(UTexture2D* InPortrait);
```

### Animation Control

```cpp
// Start typewriter animation with a preset
UFUNCTION(BlueprintCallable, Category = "Dialog")
void StartTyping(
    FName InPresetName,        // Animation preset name
    float InDuration = 1.0f,   // Animation duration (seconds)
    float InDelay = 0.0f       // Start delay (seconds)
);

// Skip the typewriter animation and show all text
UFUNCTION(BlueprintCallable, Category = "Dialog")
void SkipToEnd();

// Check if animation is currently playing
UFUNCTION(BlueprintPure, Category = "Dialog")
bool IsTyping() const;
```

### Delegates

| Delegate | Signature | Description |
| --- | --- | --- |
| `OnDialogComplete` | `()` | Triggered when the typewriter animation completes (Blueprint bindable) |

## Usage Patterns

### Basic Flow

1. Create a Blueprint subclass of `UDialogWidget`
2. Lay out UI elements in UMG Designer (text block, portrait frame, name bar, etc.)
3. In game logic, call `SetDialogText()` + `SetSpeakerName()` + `SetSpeakerPortrait()`
4. Call `StartTyping()` to begin the character-by-character animation
5. Listen for the `OnDialogComplete` event to advance the narrative after dialog finishes

### Interaction Modes

| Interaction | Behavior |
| --- | --- |
| Wait for animation completion | Auto-advance; broadcast `OnDialogComplete` when finished |
| Press confirm key | Call `SkipToEnd()` to instantly show all text |
| Press skip key | Broadcast completion event and load next dialog segment |

## Internal Implementation

`UDialogWidget` lazily creates a `UTextAnimator` instance via `GetOrCreateAnimator()`:

```
SetDialogText()
    │
    ▼
GetOrCreateAnimator()
    │
    ▼
Animator->SetRichText(RenderedRichText)
    │
    ▼
StartTyping()
    │
    ▼
Animator->StartAnimating()
    │
    ▼ (per Tick)
Animator->TickAnimation(DeltaTime)
    │
    ▼ (on completion)
OnAnimationFinished() → OnDialogComplete.Broadcast()
```

### Rich Text Rendering

The `InRichTags` content is merged with `InText` and passed to the animator via `SetRichText()`. The animator uses `FTagParser` to parse all tags and `FRenderTreeBuilder` to construct the render tree. Animation tags are stripped, while remaining standard rich text style tags are preserved in `DisplayRichText` for `SAnimatedRichTextBlock` rendering.
