> 适用版本：Astro v6.4.2 · TypeScript 5.8.3 · Tailwind CSS 4.3
> 设计语言：Material Design 3 + 抽象拟物（Abstract Skeuomorphism）混合
> 规范蓝本：`/spec/` 设计规范预览页

## 1. 设计哲学

### 1.1 抽象拟物风格

本项目采用 **Material Design 3（M3）** 与 **抽象拟物（Abstract Skeuomorphism）** 的混合设计语言。

**核心思想**：将软件中的虚拟实体对应到现实生活中的真实实体进行抽象化提取——或者说，是虚拟实体真正在现实中可能呈现的样式。

具体而言，抽象拟物是 **扁平风格** 与 **液态玻璃质感** 的结合体：

- UI 表面平整、整洁干净
- 组件不透明，但具有明确的体积感和观感上的"厚度"
- 整体效果类似于**不透明磨砂亚克力板材质**——每个组件都是一块写了字的"亚克力板"
- 通过高光、反射和阴影塑造立体感，不能完全扁平

**M3 骨架**：色彩系统遵循 M3 的 Color Role 体系，形状体系遵循 M3 Shape Scale，提供系统化的层级感知。

**Tailwind 工具层**：CSS 变量通过 `@theme` 注册到 Tailwind 4.3（`src/styles/global.css`），所有设计 Token 均可作为 Tailwind 工具类使用（如 `text-primary`、`rounded-md`、`shadow-far`）。组件中的 flex 布局、间距、字号等通用样式优先使用 Tailwind；拟物质感（噪点、color-mix、多层阴影、clip-path）保留手写 CSS。

**暗色优先**：默认主题为暗色（`:root` 即暗色 Token），亮色通过 `@media (prefers-color-scheme: light)` 和手动 `:root.light-style` 类覆盖。

### 1.2 设计原则

| 原则 | 说明 |
| --- | --- |
| **体积感优先** | 所有组件必须具备观感上的"厚度"——通过多层阴影、切边、内嵌暗边实现亚克力板的物理体积 |
| **强交互动画** | 拟物风格要求动画必须具备交互感：按钮按下沉入、hover 抬起、滑块在凹槽中物理滑动 |
| **不透明亚克力板** | 组件为实色基底 + SVG 噪点纹理 + 漫反射高光，不做毛玻璃半透明（暗色/亮色统一使用实色） |
| **多语言包容** | 10 种语言支持，阿拉伯语 RTL 方向适配，locale 感知路由 |

### 2.2 主题切换

由 `src/scripts/theme.ts` 驱动，三层优先级：

1. `localStorage.getItem('theme')` — 用户手动选择（最高优先）
2. `prefers-color-scheme` — 系统偏好
3. 默认暗色 — 无偏好时的回退

切换时使用 **View Transition API** 实现圆形辐射动画——从鼠标点击位置向外扩散展开新主题。

## 3. 字体与排版

### 3.1 字体栈

| 字体 | CSS font-family 名称 | 来源 | 用途 |
| --- | --- | --- | --- |
| MiSans Latin | `MiSansLatin` | jsDelivr CDN（`misans@4.0.0/lib/Latin/`） | 西文正文 — 单文件 woff2，专为 Latin 字符优化 |
| MiSans Normal | `MiSans` | jsDelivr CDN（`misans@4.0.0/lib/Normal/`） | CJK 回退 — Unicode 分片加载，覆盖中日韩全字符集 |
| System UI | `system-ui, -apple-system, sans-serif` | 系统内置 | 最终回退 |

```css
@import url('https://cdn.jsdelivr.net/npm/misans@4.0.0/lib/Latin/MiSansLatin-Medium.min.css');
@import url('https://cdn.jsdelivr.net/npm/misans@4.0.0/lib/Latin/MiSansLatin-Bold.min.css');

/* Normal 变体 — CJK 回退 */
@import url('https://cdn.jsdelivr.net/npm/misans@4.0.0/lib/Normal/MiSans-Regular.min.css');
@import url('https://cdn.jsdelivr.net/npm/misans@4.0.0/lib/Normal/MiSans-Medium.min.css');
@import url('https://cdn.jsdelivr.net/npm/misans@4.0.0/lib/Normal/MiSans-Bold.min.css');
```

**字体栈变量**（`src/styles/tokens.css`）：

```css
--font-sans: 'MiSansLatin', 'MiSans', system-ui, -apple-system, sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', Consolas, monospace;
```

> **字体选择逻辑**：浏览器优先使用 `MiSansLatin` 渲染西文字符（A-Z、数字、标点），遇到 CJK 字符时 MiSansLatin 无对应字形，自动回退到 `MiSans` Normal 变体。最终回退链 `system-ui` 确保任何漏网字符都有系统字体兜底。

#### 3.1.1 字重对齐

MiSans 使用 CSS Fonts Level 4 非标准权重值。为保证浏览器精确命中 `@font-face` 声明（不触发合成加粗），CSS 字重 Token 必须与 MiSans CDN 文件的声明值匹配：

| 设计语义 | CSS Token | 标准值 | MiSans 实际值 |
| --- | --- | --- | --- |
| Regular | `--font-weight-regular` | 400 | **330** |
| Medium | `--font-weight-medium` | 500 | **380** |
| Bold | `--font-weight-bold` | 700 | **630** |

> **为什么不用 400/700**：浏览器请求 `font-weight: 400` 时，MiSans 只有 weight 330 的声明。70 单位的落差会触发 DirectWrite 合成加粗——在 Windows 上严重劣化字形边缘质量。对齐到 330/380/630 后，浏览器直接匹配，无需合成。

> **`<strong>` / `<b>` 显式字重**：浏览器默认 `font-weight: bold`（=700）无法命中 MiSans 的 630 声明，同样触发合成加粗。必须在 CSS 中显式设置 `strong, b { font-weight: var(--font-weight-bold) }`（定义于 `src/styles/base.css`）。

#### 3.1.2 抗锯齿策略

| 平台 | CSS 属性 | 值 | 策略 |
| --- | --- | --- | --- |
| Windows / macOS | `-webkit-font-smoothing` | `antialiased` | 灰度抗锯齿 |
| Firefox macOS | `-moz-osx-font-smoothing` | `grayscale` | 灰度 |
| 全平台 | `text-rendering` | `optimizeLegibility` | 优先连字与字距 |

> **强制灰度的依据**：HiDPI 屏幕（缩放 ≥125%）上，子像素渲染（ClearType）会造成 MiSans 中文笔画粗细明显不均、边缘出现彩色拖影。灰度抗锯齿消除子像素色散，笔画均匀一致。此决策参考 HMCL 的 MiSans 渲染实践。

### 3.4 内联代码

行内 `code` 采用与 `.tag` 同款的磨砂亚克力板材质——不透明实色基底 + SVG FractalNoise 噪点纹理 + 160deg 漫反射高光：

- **暗色**：`background: var(--clr-primary-bg)` 实色深灰 + 灰度 SVG 噪点 × overlay(contrast 5.0) + 左上入射高光
- **亮色**：暖黄基底（`bg 97% + accent 3%`）+ SVG 噪点 tint 着色为暖黄色系（R≈G 偏高、B 压低） + 纯白强高光
- 仅左上边框 `1px solid` 模拟亚克力厚度
- `position: relative; top: -2px; left: -1px` 微调基线对齐
- 与 `.tag` 同款三层阴影结构

代码块内的 `code`（`pre code`）重置为 `position: static`，清除所有内联增强样式。

### 5.2 亚克力板阴影体系

组件采用 **三层阴影结构** 模拟亚克力板的物理体积（详见 §6.1）：

| 层 | 作用 | 形态 |
| --- | --- | --- |
| L1 切边厚度 | 板边自身的物理厚度 | 无模糊硬边偏移（如 `4px 4px 0 0`） |

## 6. 抽象拟物风格

### 6.1 核心材质：不透明磨砂亚克力板

所有组件采用统一材质体系——**实色基底 + SVG FractalNoise 噪点纹理 + 漫反射高光**。

**物理模型**：光源从偏上方（160deg）入射，在亚克力板表面产生以下效果：

```
光源方向：偏上方入射（160deg）
         ▲ 高光（::after, 左上最亮）
  ┌──────┐
  │ ╲    │ ← 左上内嵌暗边 (L3, 光被板体吸收)
  │  ╲   │
  │   ╲  │ ← 右下切边聚光 (L1, 光穿透板体在切边散射)
  └──────┘
       ██ ← 右下外露厚度 (亮 cut edge)
  ░░░░░░░░ ← 环境柔影 (L2, 偏右下)
```

**暗色/亮色技术路线差异**：

|  | 暗色模式 | 亮色模式 |
| --- | --- | --- |
| 基底 | 实色深灰（`--clr-primary-bg`） | 实色暖黄（`bg 97% + accent 3%`） |
| 噪点 | SVG 灰度噪点 × `overlay` 混合 | SVG 噪点直接在滤镜中着色为黄色系（无需外部混合） |
| 高光 | 白色微量渐变（opacity 8%） | 白色强高光（opacity 75%） |
| 原则 | 低对比度克制 | 人眼对浅色 alpha 不敏感 → 实色全量覆盖 |

### 6.2 材质变量

| CSS 变量 | 暗色值 | 作用 |
| --- | --- | --- |
| `--edge-highlight` | `inset 0 1px 0 rgba(255,255,255,0.08)` | 顶边高光 |
| `--edge-shadow` | `inset 0 -1px 0 rgba(0,0,0,0.12)` | 底边阴影 |
| `--groove-bg` | `base-bg 60% + primary-bg` 混合 | 凹槽轨道背景 |
| `--groove-inset` | `inset 0 1px 3px rgba(0,0,0,0.25)` | 凹槽内嵌阴影 |

### 6.3 凹槽滑动机制

导航菜单切换采用**凹槽滑动**模式——一个物理亚克力滑块在凹槽轨道中滑动，滑动到的位置变色。

**实现架构**：

```
ul (凹槽轨道, isolation: isolate)
  ├─ ::before SVG 噪点            z-index: 1
  ├─ .nav-slider (滑块)            z-index: 2
  │    └─ .nav-slider-highlight     z-index: 3
  ├─ a (未选中, translate(2px,2px))  z-index: 3
  ├─ a.active (选中, -5px 抬起)     z-index: 20
  └─ a.nav-lift (悬浮)              z-index: 22
```

**核心原则**：滑块始终在所有文字之下（`z-index: 2`），文字永不被遮挡。

**滑块双层抬起**：

| 状态 | slider Y | lifted class | 高光 opacity |
| --- | --- | --- | --- |
| 默认 | `translateY(-2px)` | 无 | 0.5 |
| hover | `translateY(-5px)` | `.lifted` | 1 |
| 按下 | `translateY(0)` | 移除 | 0.5 |

**文字 CSS class 体系**（纯 class 驱动，无 inline transform 冲突）：

| Class | Transform | Transition | 场景 |
| --- | --- | --- | --- |
| base `a` | `translate(2px, 2px)` | 0.2s spring | 未选中，沉入凹槽 |
| `.nav-lift` | `translate(1px, -1px)` | 0.2s 0.35s spring | 滑块到达，默认浮起 |
| `.nav-hover-lift` | `translate(2px, -3px)` | 0.2s 0.15s spring | hover 普通标签 |
| `.nav-hover-lift.instant-hover` | `translate(1px, -5px)` | 0.2s 0.05s spring | hover active（直接） |
| `.nav-hover-lift.instant-return` | `translate(1px, -5px)` | 0.2s 0.15s spring | 从别处回到 active |
| `.nav-pressed` | `translate(2px, 2px)` | 0.08s ease | 按下普通标签 |
| `.nav-pressed.instant-press` | `translate(1px, 1px)` | 0.08s ease | 按下 active |

**交互场景时序**：

| 触发 | 滑块动作 | 文字效果 |
| --- | --- | --- |
| 页面加载 | 瞬间定位到 active，-2px | active 加 `.nav-lift`（无动画） |
| Hover active | 无滑动，升至 -5px + lifted | `.instant-hover`：0.05s delay → 浮起 |
| Hover 普通标签 | 滑到目标，升至 -5px + lifted | `.nav-hover-lift`：0.15s delay → 浮起 |
| 回到 active | 滑回 active，升至 -5px + lifted | `.instant-return`：0.15s delay → 浮起 |
| Mouse leave | 150ms 后滑回 active，-2px | 文字立即沉（`delay: 0s !important`） |
| 按下 | 滑块降至 0，移除 lifted | `.nav-pressed`：0.08s 快压 |
| 松手 | `transition: none` 瞬回 | 同瞬回 |

**缓动曲线**：

- 滑块 `left`/`width`：`cubic-bezier(0.4, 0, 0.2, 1)`（零 overshoot，凹槽约束）
- 文字 `transform`：`cubic-bezier(0.34, 1.3, 0.64, 1)`（允许微 overshoot）
- 高光 `opacity`：`ease 0.4s`

**实现文件**：

- 文字 class 样式 + 滑块样式：`src/styles/ui-spec.css`
- 交互逻辑（创建滑块、事件绑定、recoverPress）：`src/scripts/nav-slider.ts`

### 7.3 卡片 `.card`

定义于 `src/styles/ui-spec.css`。**/spec/ 规范预览页的视觉方案（实色亚克力板）为最终规范。**

> 注：`components.css` 中存在另一套毛玻璃方案（`backdrop-filter: blur()` + 半透明基底），该方案仅用于普通页面。本文档描述的是 /spec/ 页面使用的**实色亚克力板方案**。

#### 通用属性

| 属性 | 值 | 说明 |
| --- | --- | --- |
| `position` | `relative` | 伪元素定位锚点 |
| `border-radius` | `--shape-md` (12px) | 统一圆角 |
| `padding` | `var(--space-xl)` (32px) | 内边距 |
| `overflow` | `hidden` | 裁剪伪元素（有丝带时改为 `visible`） |
| `isolation` | `isolate` | 独立层叠上下文 |
| `backdrop-filter` | `none` | 不使用毛玻璃 |

**内容保护**：`.card > * { position: relative; z-index: 3; }` — 确保子元素在噪点和反光层之上。

#### 暗色模式

**基底与边框**：

- `background`: `var(--clr-primary-bg)` 实色深灰
- `border-top/left`: `1px solid color-mix(text 5%, transparent)` 微光边框
- 右/下无边框（由 box-shadow 切边替代）

**三层阴影**：

- L1：`4px 4px 0 0 color-mix(text 20%, transparent)` — 右下切边聚光
- L2：`10px 8px 6px 1px rgba(0,0,0,0.30)` — 环境柔影
- L3：`inset 5px 5px 3px 0 rgba(0,0,0,0.15)` — 左上内嵌暗边

**磨砂噪点 `::before`（z-index: 1）**：SVG feTurbulence (baseFrequency=0.19, 3 octaves) → 灰度平均化 → `contrast(8.0)` + `mix-blend-mode: overlay` + CSS opacity 0.25

**漫反射高光 `::after`（z-index: 2）**：`linear-gradient(160deg, color-mix(text 8%, transparent) 0%, transparent 45%)`

#### 亮色模式

- 基底：`color-mix(bg 97%, accent 3%)` 暖黄白
- 边框：`color-mix(bg 80%, accent 20%)`
- L1 切边：`color-mix(bg 94%, accent 6%)`
- L3 内嵌：`color-mix(bg 65%, accent 35%)`
- 噪点：SVG 直接在滤镜中 tint 着色为黄色系（`mix-blend-mode: normal`）
- 高光：`rgba(255,255,255,0.75)` 纯白强高光

双路径驱动：`:root.light-style .card` + `@media (prefers-color-scheme: light) { .card }`，两路规则等价。

#### Card 弹性布局系统

Card 内部使用 **Tailwind CSS** 工具类提供弹性布局骨架，替代原手写 CSS：

| 作用 | Tailwind 类 | 说明 |
| --- | --- | --- |
| 弹性列骨架 | `flex flex-col gap-0` | 原 `.card-layout` |
| 图标+标题行 | `flex items-center gap-2` | 原 `.card-header-row` |
| 可伸缩内容区 | `card-body` 类（见下方） | `flex: 1; margin: 0` |
| 底部标签区 | `flex gap-2 flex-wrap items-center self-start pb-0.5` | 原 `.card-footer` |

**Card Body 间距规范**（scoped CSS，`src/pages/spec/index.astro`）：

```css
.card-body {
    flex: 1;
    margin: 0;
    padding: var(--space-sm) 0 var(--space-md); /* 上8px 下16px */
}
.card-body-featured {
    padding: var(--space-md) 0 var(--space-sm); /* 特色模块：上16px 下8px */
}
```

- `.card-body`：通用卡片正文区（适用于最新文章等场景）
- `.card-body-featured`：特色模块专属变体，缩小下间距为丝带留出紧凑呼吸空间

**卡片内嵌 Icon**（`.card-icon-embed`，debossed 刻入效果）：

使用 `text-shadow` 模拟文字刻入卡面的凹槽感——光源左上 160deg，上壁投暗影、下壁反亮光：

```css
.card-icon-embed {
    text-shadow:
        -2px -2px 0 rgba(0, 0, 0, 0.33),
        1.5px 1.5px 0 rgba(255, 255, 255, 0.06);
    transform: translate(-1px, -1px);
}
:root.light-style .card-icon-embed {
    text-shadow:
        -1px -1px 0 color-mix(in srgb, var(--md-sys-color-primary), #000000 22%),
        1.5px 1.5px 0 rgba(255, 255, 255, 1);
    transform: translate(-1px, -1px);
    opacity: 0.8;
}
```

亮色模式下暗影使用 `color-mix(主强调色, 黑 22%)` 而非纯黑，与暖色基底融合更自然。

#### 折叠丝带 `.card-ribbon`

基于 freeCodeCamp 纯 CSS 折叠丝带技术（`clip-path` 裁切右下折角 + `inset box-shadow` 模拟折痕阴影）。丝带作为 `.card` 的直接子元素，`position: absolute` 定位在卡片右下角。

**几何参数**：右伸出量 `--f: 10px`，板边厚度补偿 `--c: -4.5px`，折角深度 `--d: 5.5px`，左侧箭头 `--r: 14px`，距底上移 `--y: 8px`。

**交互状态**：

| 状态 | 丝带 | 阴影 | 文字 `<span>` |
| --- | --- | --- | --- |
| 基态 | 默认 padding | 默认 | 无位移 |
| Hover | `padding-left` +18px（左伸） | `padding-left` 同步 +18px | `translateX(-18px)` 抵消，净位不变 |
| Active | `padding-left` +36px + `rotate(-4deg)` | 同 +36px + rotate(-4deg) | `translateX(-36px)` 抵消 |

丝带使用独立阴影元素 `.card-ribbon-shadow`（`z-index: 5`，低于丝带 `z=10`），暗色 `rgba(0,0,0,0.42)`、亮色 `rgba(0,0,0,0.18)`，`filter: blur(1.5px)` 柔化。

**丝带高光体系**：`::before`（160deg 白色渐变）+ `::after`（菲涅尔反射——上边 `1px rgba(255,255,255,0.16)` + 左边 `0.08`），暗色/亮色统一白色文字 `#FFFFFF`。

#### Card 内嵌标签 `.card .tag`

当 `.tag` 出现在 `.card` 内部时切换为**下凹/内嵌样式**——标签是 Card 表面压出的凹槽而非凸起的亚克力板：

- 背景更暗（`bg 50% + base` ≈ `#1c1c1c`）
- 四边暗框（`border-top/left: text 10%` / `border-right/bottom: text 15%`）
- 内嵌硬边 `inset 2px 3px rgba(0,0,0,0.45)` + 右下散射反光 `inset -1px -2px 6px text 10%`
- 文字 `translate(1px, 2px)` 向右下偏移模拟凹槽透视
- hover 时从凹槽中凸起（强调色 Container 背景 + 抬起阴影），active 时向右下按入

亮色版：基底 `bg 67% + accent 33%`，右下反光 `rgba(255,255,255,0.80)` 纯白强散射。

### 7.4 按钮 `.btn`

| h2 | `--font-size-3xl` (2rem) | bold 700 | `32px 0 16px` | `--md-sys-color-on-surface` |

所有标题统一设置 `scroll-margin-top: calc(var(--header-height) + var(--space-lg))`，确保锚点跳转时不被固定导航栏遮挡。

- 正文段落：`margin-bottom: var(--space-md)`，行高 `--line-height-relaxed` (1.75)

**列表**：

- **无序列表 `ul`**：`list-style-type: disc`，`padding-left: var(--space-xl)`。列表项 `li`：`margin-bottom: var(--space-sm)`，`line-height: var(--line-height-relaxed)`（1.75）。内嵌段落末尾 `margin-bottom: 0` 消除多余空行
- **有序列表 `ol`（顶层步骤）**：采用 **步骤数字圆圈 + 连接竖线** 样式。数字圆圈使用页头按钮同款亚克力板材质（`color-mix` 强调色基底 + SVG 噪点 + 160deg 高光 + 三层阴影），32px 圆形，`counter(steps)` 编号，白色文字。竖线 `::before` 2px `outline-variant` 色，从圆圈下方延伸至步骤底部。亮色模式下数字圆圈切换为纯强调色基底 + 白色噪点 + 纯白高光。末尾子元素 `margin-bottom: 0` 消除多余空行
- **内嵌有序列表 `ol ol` / `ul ol`**：恢复默认 `list-style-type: decimal`，`padding-left: var(--space-xl)`。列表项 `li`：`padding-left: var(--space-sm)`（序号与文本 8px 间距），`margin-bottom: var(--space-sm)`，`::marker { font-weight: bold }` 仅序号加粗。清除所有伪元素（无圆圈/竖线）

**列表内嵌段落**：Markdown 中缩进内容前需要空行才会生成独立 `<p>`（否则渲染为同行续写）。步骤内嵌段落的相邻子元素间保持 `margin-top: var(--space-sm)` 间距。

**分隔线 `hr`**：`border-top: 3px solid color-mix(in srgb, var(--clr-primary-text) 80%, transparent)`，上下 `margin: var(--space-2xl) 0`。

**文章配图 `.article-content img`**：

- `display: block; max-width: 100%; height: auto; margin: 0 auto`
- `border-radius: var(--shape-md)` + 悬浮阴影（暗色：`0 2px 8px rgba(0,0,0,0.20), 0 8px 24px rgba(0,0,0,0.15), 0 16px 48px rgba(0,0,0,0.08)`；亮色：同结构但 opacity 降低至 0.05/0.04/0.02）
- 独图段落（`p:has(> img:only-child)`）：`display: inline-block; background: var(--clr-base-bg); border-radius: var(--shape-md)` + `::after` 纸张噪点叠加层（SVG feTurbulence + 155deg 漫反射打光）。亮色版高光切换为白色强高光，噪点 tint 着色为暖黄色系

**图片纸张噪点（`<picture>` 容器）**：通过 `picture::after` 在 `<picture>` 容器上叠加极淡 SVG feTurbulence 噪点纹理——模拟纸面质感。
- `picture` 设置为 `display: block; position: relative; overflow: hidden; border-radius: var(--shape-md)`
- `picture::after`：`z-index: 2`（图像上方），`mix-blend-mode: overlay; opacity: 0.08`
- SVG 参数：`baseFrequency=0.28`，`numOctaves=3`，`background-size: 70px 70px`

#### 7.7.1 图片输出格式

所有内容集合（Content Collection）图片通过自定义图片服务 `src/scripts/avif-image-service.mjs` 输出为 **AVIF 格式**（替代 Astro 默认的 WebP）。该服务包装 `astro/assets/services/sharp`，在 `validateOptions` 中注入 `format: 'avif'`。

**实现方式**：
- 自定义服务注入 `format='avif'` 到 Sharp 编码器
- 开发服务器通过 `/_image` 动态端点实时渲染
- 生产构建输出 `/_astro/xxx.hash.avif` 静态文件
- 保留 `loading="lazy"`、`decoding="async"`、`width`/`height` 属性

**配置**（`astro.config.mjs`）：
```js
image: {
    service: {
        entrypoint: './src/scripts/avif-image-service.mjs',
    },
},
```

> 源文件为 AVIF 格式时，Sharp 直接以 AVIF 编码输出（无需二次转码），压缩率比 WebP 高约 40%。

#### 7.7.2 图片点击放大（Lightbox）

文档页和博客正文中的图片支持单击全屏放大覆盖：

- **触发**：单击 `.article-content img` 元素
- **效果**：85% 不透明度黑色覆盖层 + 图片居中（max 90vw×90vh），阴影 `0 8px 40px rgba(0,0,0,.5)`，圆角 8px
- **关闭**：点击覆盖层或按 `Escape` 键
- **实现**：`ProjectDoc.astro` 和 `BlogLayout.astro` 中的客户端 `<script>` 块，动态创建 overlay `<div>` 和 `<img>` 元素
- **光标**：`.article-content img { cursor: zoom-in }`（定义于 `components.css`）
- **不依赖外部库**：纯 DOM API 实现，无 JS 依赖

#### 7.7.3 有序列表缩进对齐

## 13. 文档系统架构

### 13.1 概述

项目文档系统（Project Docs）为每个项目提供多章节、多页面的文档支持。每个项目可包含一个文档树，支持 10 种语言翻译、自动生成导航注册表、章节分组侧边栏和跨文档前后篇导航。

### 13.2 文件结构

```
src/content/
└── projects/                       ← 所有项目内容
    └── {slug}/                     ← 项目标识符
        ├── index.md                ← 文档首页，H1 = 项目标题，首行 = 项目摘录（zh-cn）
        ├── index.{locale}.md       ← 文档首页翻译
        ├── {NN}.{chapter-slug}/    ← 章节文件夹
        │   ├── {NN}.{doc-slug}.md         ← 文档（zh-cn）
        │   ├── {NN}.{doc-slug}.{locale}.md ← 文档翻译
        │   └── ...
        └── ...
```

**命名约定：**
- `{slug}` — 项目标识符，小写 kebab-case，同时也是翻译键前缀
- `{NN}` — 两位数字排序前缀（`01`、`02`…`99`）
- `{chapter-slug}` — 章节名，kebab-case，与 slug 共同构成翻译键 `{slug}-{chapter-slug}`
- `{doc-slug}` — 文档名，kebab-case
- `{locale}` — 语言代码（`en-us`、`ja-jp` 等），默认语言 zh-cn 省略后缀

### 13.3 文档注册表

编译时自动生成文档注册表（Document Registry），无需手动维护索引文件。注册表结构如下：

```typescript
interface DocRegistryEntry {
  projectSlug: string;       // 项目标识符
  id: string;                // 完整文档 ID（含路径）
  title: string;             // 文档标题（来自 H1）
  locale: string;            // 语言代码
  chapterSlug: string;       // 章节标识符
  chapterOrder: number;      // 章节排序（01、02…）
  docOrder: number;          // 文档排序（01、02…）
}

interface DocRegistryChapter {
  slug: string;              // 章节标识符
  order: number;             // 章节排序
  titleKey: string;          // 翻译键（{slug}-{chapter-slug}）
  docs: DocRegistryEntry[];  // 章节内文档列表
}
```

注册表特性：
- **扁平列表**：所有文档按 `(chapterOrder, docOrder)` 排序，用于前后篇导航
- **分组索引**：按 chapter 分组，用于侧边栏渲染
- **翻译感知**：每个文档的 locale 字段用于按语言筛选
- **构建时生成**：通过 Astro 自定义 Loader 在 `content.config.ts` 中生成

### 13.4 翻译机制

- **翻译键**：章节标题翻译键为 `{slug}-{chapter-slug}`（如 `texturge-getting-started`），在 i18n JSON 字典中定义
- **文档标题**：直接取自 Markdown 文件中的 H1，不通过翻译键
- **locale 后缀**：文件名以 `.{locale}.md` 结尾表示翻译版本，默认语言不加后缀
- **回退策略**：未提供翻译的语言自动回退至 zh-cn 版本

### 13.9 本页目录（TOC）凹槽滑块

右侧栏 TOC 将 H2/H3 标题按 H2 分组，每个 H2 为一个 `.doc-toc-group` 容器。每个 group 内嵌一个与语言下拉栏同款的**凹槽滑块**（`.doc-toc-slider`），默认定位在 H2 上，hover 到 H3 时滑块移入 H3，离开后回到 H2。

**架构**：

```
.doc-toc-group.has-toc-slider (内嵌凹槽, isolation: isolate)
  ├─ ::before SVG 噪点                    z-index: 1
  ├─ .doc-toc-slider (滑块，H2 accent 材质)  z-index: 2
  │    └─ .doc-toc-slider-highlight            z-index: 3
  ├─ a.doc-toc-link.doc-toc-depth-2            z-index: 3
  └─ a.doc-toc-link.doc-toc-depth-3            z-index: 3
```

**容器 `.doc-toc-nav`**：`display: flex; flex-direction: column; gap: var(--space-sm)`。标题 `.doc-toc-label`：`font-size: var(--font-size-lg)`，`font-weight: bold`，`text-transform: uppercase`，`letter-spacing: 0.06em`。

**内容保护**：`.doc-toc-group > * { position: relative; z-index: 3 }`。

**内嵌分组 `.doc-toc-group`**：内边距 `var(--space-sm)`（8px），圆角 `var(--shape-lg)`（24px）。暗色/亮色内嵌凹槽材质同语言下拉栏 `ul`。`position: relative`，单 H2 也初始化滑块。

**暗色模式**：

| 属性 | 值 |
| --- | --- |
| 背景 | `color-mix(primary-bg, base-bg 50%)` |
| 四边边框 | `border-top/left: text 10%` / `border-right/bottom: text 15%` |
| L1 外投影 | `2px 4px 3px 0 text 20%` |
| L2 内嵌硬边 | `inset 4px 6px 0 0 rgba(0,0,0,0.45)` |
| L3 内嵌反光 | `inset -2px -3px 8px text 10%` |
| 噪点 `::before` | SVG turb (0.19/3) + contrast(5.0) + overlay, opacity 0.33, z-index: 1 |

**亮色模式**：基底 `color-mix(bg, accent 33%)`，边框 `bg+accent 20%` 四边 + 右下 `rgba(255,255,255,0.15)` 反光，阴影 `bg 94%+accent 6%` 外投影 + `accent+白 10%` 内嵌 + `rgba(255,255,255,0.80)` 散射。噪点 SVG tint 着色为暖色系，`mix-blend-mode: normal`，opacity 1。

**`.doc-toc-slider` 滑块材质**：与 `.doc-toc-depth-2` 同款 accent 亚克力板材质——accent 实色基底 + SVG tint 着色噪点 + 160deg 漫反射高光。暗色切边 `2px 4px 0 0`、亮色 `1px 4px 0 0`。

**滑块 CSS transition**：
```css
transition:
    top 0.4s cubic-bezier(0.4, 0, 0.2, 1),
    height 0.3s cubic-bezier(0.4, 0, 0.2, 1),
    transform 0.25s cubic-bezier(0.4, 0, 0.2, 1),
    box-shadow 0.4s ease,
    border-color 0.4s ease;
```

滑块位置使用 `offsetTop`/`offsetHeight` 计算（而非 `getBoundingClientRect`），避免父级 border 造成的定位偏差。

**滑块按下 transition**（`mousedown`）：
```css
transform 0.06s ease, box-shadow 0.1s ease, border-color 0.1s ease
```
按下时滑块快速降至 `translateY(0)`，松手时通过 `slider.style.transition = 'none'` 瞬间定位。

**mouse leave 防抖**：离开 `.doc-toc-group` 后 150ms 延迟才执行回落（`setTextHover(false)` + `setSlider(false)` + `lift(h2Link)` + `positionAt(h2Link, true)`），期间重新进入则取消。文字回落带 `transform 0.2s 0.05s` 动画，滑块回落同时执行 `transform` drop（0.25s）和 `top` 动画（0.4s）。

**切换元素时**：hover 到新链接前，先清空旧链接的 inline `transform` 和 `transition`，避免旧链接停留在抬起位置。

**滑块高光体系**：

| 层 | 暗色 | 亮色 | 作用 |
| --- | --- | --- | --- |
| `::after` 基础反射 | `text 12% → 45%`（常量） | `rgba(255,255,255,0.15) → 45%` | 基态常驻反射 |
| `.doc-toc-slider-highlight` | `text 40% → 55%`，opacity 0.5→1 | `rgba(255,255,255,0.5) → 55%`，opacity 0.5→1 | 交互高光层 |

**滑块双层抬起**：

| 状态 | slider Y | lifted class | 高光 opacity |
| --- | --- | --- | --- |
| 默认 | `translateY(0)` | 无 | 0.5 |
| hover | `translate(-1px, -6px)` | `.lifted` | 1 |
| 按下 | `translateY(0)` | 移除 | 0.5 |

**④ TOC 文字 class 体系**（纯 inline style 驱动，不再依赖 CSS class transition 层叠）：

文字动画已从 CSS class 层叠迁移至 JS inline style，由 `setTextHover()` 统一管理。hover 时先用 inline style 设 `transition` 再设 `transform`，确保浏览器正确检测动画。

| 场景 | inline Transform | inline Transition | 条件 |
| --- | --- | --- | --- |
| hover 到 H2（滑块已在 H2） | `translate(-1px, -6px)` | `transform 0.2s 0.05s cubic-bezier(0.34, 1.3, 0.64, 1)` | `instant && !returning` |
| hover 到 H2（从 H3 回来） | `translate(-3px, -6px)` | `transform 0.2s 0.05s cubic-bezier(0.34, 1.3, 0.64, 1)` | `instant && returning` |
| hover 到 H3（滑块需滑过去） | `translate(-5px, -5px)` | `transform 0.2s 0.05s cubic-bezier(0.34, 1.3, 0.64, 1)` | 普通 hover |
| hover 离开（回落） | `''`（清空） | `transform 0.2s 0.05s cubic-bezier(0.34, 1.3, 0.64, 1)` | `setTextHover(false)` |
| 按下（instantMode） | `translate(-1px, 1px)` | `transform 0.08s ease` | `mousedown` + `instantMode` |
| 按下（非 instantMode） | `translate(-2px, 1px)` | `transform 0.08s ease` | `mousedown` |
| 松手（仍在 hover）恢复 hover 位 | `translate(-1px, -6px)` / `translate(-3px, -6px)` | `transform 0.0s ease`（无动画） | `mouseup` + `hovering` |

**交互场景时序**：

| 触发 | 滑块动作 | same | returning | 文字效果 |
| --- | --- | --- | --- | --- |
| 页面加载 | 瞬间定位到 H2，`translateY(0)` | — | — | H2 加 `.toc-lift` class 白字（无动画） |
| hover 到 H3 | 滑到目标，升至 `translate(-1px, -6px)` + lifted | false | false | `.toc-hover-lift` class + inline `translate(-5px, -5px)` 0.05s delay |
| hover 到 H2 | 无滑动，升至 `translate(-1px, -6px)` + lifted | true | false | inline `translate(-1px, -6px)` 0.05s delay |
| 从 H3 回到 H2 | 滑回 H2，升至 `translate(-1px, -6px)` + lifted | true | true | inline `translate(-3px, -6px)` 0.05s delay |
| mouse leave | 150ms 后滑回 H2（`translateY(0)` + top 动画到 H2） | — | — | inline `transform` 清空 + 0.05s delay transition 平滑回落 |
| 按下 | 滑块 `translateY(0)` 0.06s，移除 lifted | — | — | inline `translate(-1px, 1px)` / `translate(-2px, 1px)` 0.08s ease |
| 松手（仍在 hover） | 滑块 `translate(-1px, -6px)` 瞬回 + lifted | — | — | inline 恢复 hover 位 `transform 0.0s ease`（无动画瞬回） |
| 松手（不 hover） | 滑块 `translateY(0)` 瞬回 | — | — | |

**实现文件**：

- 视觉样式（凹槽、滑块、H2 材质、文字 class、亮色覆盖）：`src/styles/doc-shared.css`
- 交互逻辑（滑块创建、hover/leave 事件绑定、instant 判定、pressed 恢复）：`src/scripts/toc-slider.ts`

## 14. 无障碍设计

- **颜色对比度**：暗色主文本 `#F5F5F5` vs 底背景 `#141414` 对比度 ≥ 4.5:1（AA 级）

- **屏幕阅读器**：`.sr-only` 类（`position: absolute; width: 1px; ...`）隐藏视觉元素但保留读屏
- **语义化 HTML**：`<header>` / `<nav>` / `<main>` / `<footer>` / `<article>` / `<section aria-label="...">`
- **语言标记**：`<html lang="{locale}" dir="ltr|rtl">`，阿拉伯语 RTL 方向
- **触控目标**：按钮、导航链接最小 44×44px
- **键盘导航**：Tab 键自然流经导航链接 → 主题切换 → 语言切换，Escape 关闭下拉菜单
